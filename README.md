## Структура - как хранить домашки?

### Создаём структуру для сдачи пошагово

1. В этом репозитории создайте один Unity проект. Можно создать в корне репозитория, можно создать подпапку.
2. Создайте в проекте папку для темы, над которой вы работаете. Например, Coroutines
3. Если в теме несколько заданий - создайте попапку на каждое задание. Например, Task-1, Task-2
4. В подпапке создайте папку с названием Logic. Обязательно именно таким. Храните в ней скрипты, относящиеся к заданию. Подпапки в папке Logic создавать можно, если вы хотите организовать структуру (например создать папки для скриптов Infrastructure, UI, Core...)
5. Остальные файлы, относящиеся к заданию (префабы, спрайты...) храните в свободном формате.

Ключевое организационное правило: все наши скрипты обязаны находиться только в папке Logic внутри соответствующей папки задачи. Скрипты должны находиться только в папке Logic и нигде больше!

### Структура репозитория

```text
/ student-template      ← Корень репозитория. Если хочется можно - можно создать подпапку для Unity проекта
/ README.md
├── Coroutine/      ← Название темы домашки
│   ├── Task-1/         ← Задача 1 темы 'Coroutine'
│   │   ├── Logic/           ← здесь должны быть все скрипты задачи
│   │         ├── MyScript.cs
│   │         └── AnotherScriptDirectory        ← при желании можно создавать подпапки, для организации скриптов
│   │
│   ├── Task-2/         ← Задача 2 темы 'Coroutine'
│   │   ├── Logic/
│   │         └── MyScript.cs
```

- Каждая задача должна быть отдельной подпапкой в Tasks/ (например Task-01).
- Все исполняемые скрипты и исходники, относящиеся к реализации задачи, должны находиться исключительно в Tasks/<имя-задачи>/Logic/.
- В Logic/ можно хранить произвольную структуру (подпапки, модули). Главное, чтобы скрипты были только внутри этой папки.
